# Void Nexus Development Rules

## General

- Never rewrite existing files unless requested.
- Always output complete files.
- Stop at the response limit.
- Wait for "Continue".
- Never leave a file incomplete.
- Use TypeScript everywhere.
- Prefer Server Components.
- Keep code production-ready.
- Keep components reusable.
- Follow Next.js 16 App Router conventions.

## Design

- Luxury, premium, handcrafted.
- Never use generic Tailwind layouts.
- Animations must be subtle.
- Accessibility is mandatory.
- Mobile-first.
- Lighthouse target: 95+.

## Backend

- Use Supabase.
- Use Row Level Security.
- Never hardcode secrets.
- Follow the existing project architecture.
- Use existing folders.
- Respect roles and permissions.

## Workflow

1. Read existing files.
2. Generate only requested files.
3. Do not regenerate previous work.
4. Explain breaking changes before making them.
